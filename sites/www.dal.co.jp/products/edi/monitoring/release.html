EDI（企業間電子商取引）

様々な形態のB2B（企業間電子取引)を成功に導くミドルウェア製品を提供しています。

ACMS運用の安定稼働のために

[リリース情報]

  • ACMS運用監視機能バージョン1.0を2015/8/31にリリースしました

 

バージョン　1.0
ACMS運用監視リリースノート情報　2015/8/31

■トピックス

　新製品：ACMS 運用監視機能をリリースしました。

　ACMS運用監視機能は、ACMS E2X/B2Bを運用する利用者を対象とした機能です。
　通信やアプリケーションの遅延や滞留、ACMS運用担当者が予定した時刻に通信が行われない等、EDI業務を行う上でクリティカルな問題となりうる状況に対して、運用画面上で警告や状況確認等を行えるようなユーザ補助を目的としたいくつかの機能を実装しています。
　また、EDI業務上問題となりうる状況に対しては、その状況毎にACMS運用担当者に対してメールで通知を行う機能を実装しています。

■機能

  1. 運用者/一般者ID管理
    ACMSに登録されている運用者または一般者が本機能にログインする際のアクセス制御を行うことが可能です。

  2. メールサーバ設定
    メール通知を行うメールサーバを指定することができます。メール送信時に使用可能なプロトコルおよびオプションを指定することが可能です。
    ・プロトコル
    　SMTP、SMTPS
    ・オプション
    　SMTP-AUTH
    　SMTPSTARTTLS
    　SMTP-AUTH+SMTPSTARTTLS

  3. 稼働記録設定
    稼働記録ファイルの配置場所が変更された場合に一括して稼働記録監視設定の定義を変更することが可能です。

  4. 稼働記録監視設定
    稼働記録ファイルに出力したログに条件を指定してメール通知を行うことが可能です。通知先のメールアドレスや、ACMSの各定義のワイルドカート(*)指定、メールのテンプレートを選択することも可能です。
    以下のログ出力条件が指定可能です。
    ・メッセージコード
    ・タイプ
    ・運用グループ
    ・ユーザ
    ・アプリケーション
    ・ファイル
    ・回線グループ
    ・APポートグループ

  5. 通信/AP滞留監視設定
    ユーザ単位の通信トランザクションまたはユーザに紐付く回線グループ名、アプリケーション単位のAPトランザクションまたはアプリケーションに紐付くAPポートグループ単位でトランザクション滞留時に検知する際の定義を作成することが可能です。
    本機能での設定値は、通信滞留一覧またはAP滞留一覧にて画面にて確認するときの設定値およびトランザクション滞留超過時のメール通知の設定で使用されます。

  6. 通信予定設定
    通信を予定している時刻を定義することが可能です。
    本機能での設定値は、通信予実績一覧および通信予定時刻内で正常通信が行われなかった際のメール通知で使用されます。

  7. メールテンプレート編集
    メール送信時に使用するテンプレートを作成/編集することが可能です。

  8. ACMS監視設定
    ACMSの稼働監視を行うことが可能です。
    稼働が確認できない場合に、指定されたメールアドレスへの通知や監視間隔、監視有無の設定を行うことが可能です。

  9. 正常通信結果CSV出力
    1日単位で通信が正常終了した結果をCSVで出力することができます。

  10. 通信実績一括ダウンロード
    複数日の通信記録ファイルをマージし、ダウンロードすることができます。

  11. 通信滞留一覧
    通信/AP滞留監視設定にて登録した通信トランザクション滞留状況を一覧で表示します。
    通信トランザクション滞留超過時には赤く表示し、ACMS運用管理者に警告します。

  12. AP滞留一覧
    通信/AP滞留監視設定にて登録したAPトランザクション滞留状況を一覧で表示します。
    APトランザクション滞留超過時には赤く表示し、ACMS運用管理者に警告します。

  13. 回線使用状況一覧
    1日の回線の使用状況を確認することができます。
    通信トランザクション滞留が発生している場合には赤色で表現し、ACMS運用管理者に警告します。

  14. 通信予実績一覧
    通信予定設定で設定した通信予定および実際の通信実績をグラフ表示します。

  15. ACMS E2X/B2Bとの連携
    ACMSサーバを分散環境で運用されている場合、ACMS運用監視機能はACMS管理サーバと同じノードにインストールする必要があります。
    ACMS運用監視機能はACMS E2X/B2B側のトランザクションとacmsactを同期します。そのためACMSの管理サーバと同じノードで起動する必要があります。

  16. 稼働環境
    1)対応OS
    ACMS E2X/B2Bの管理サーバと同じノードにインストールする必要があり、対応OSは以下のいずれかのOS環境で利用することが可能です。
    ・Windows Server 2012 R2
    ・Windows Server 2012
    ・Windows Server 2008 R2
    ・Windows Server 2008(x64)
    ・RedHat Enterprise Linux 6(x64)

    2)Java
    ACMS運用監視機能は、以下にあるJava環境で利用できます。
    ・JDK8 / JRE8
    ・JDK7 / JRE7
    ・JDK6 / JRE6
    （推奨 JDK7）

    Linux環境
    ・OpenJDKのインストールが必要となります。
    Windows環境
    ・OracleJavaが同梱されています。

    3)DBMS
    ACMS運用監視機能には、PostgreSQL 9.1が同梱されていますが、ACMS E2X/B2Bで使用しているデータベースがPostgreSQLの場合、同一のPostgreSQLを利用することはできません。
    ACMS運用監視機能ではACMS E2X/B2Bの管理データベースにアクセスを行いますが、以下の管理データベースには対応しております。
    ・SQL Server 2014
    ・SQL Server 2012 SP1
    ・SQL Server 2008 R2 SP1 / SP2 / SP3
    ・SQL Server 2008 SP3 / SP4
    ・Oracle Database 12c(12.1)
    ・Oracle Database 11g(11.1 / 11.2)
    ・DB2 UDB 9.5 / 9.7
    ・MySQL 5.5
    ・PostgreSQL 9.0 / 9.1 / 9.2 / 9.3

    4)Web Application Server
    ACMS運用監視機能では、管理情報を登録・編集するためにWeb Application Server（以降WAS）が必要で、以下のWASを利用することが可能です。
    ・Tomcat 7.0
    ・WebSphere Application Server 7.0 / 8.5
    ・Weblogic Server 10g(10.3) / 11g(10.3) / 12c(12.1)

    5)対応ブラウザ
    ACMS運用監視機能の各管理画面の操作では、以下のブラウザを利用することが可能です。
    ・Internet Explorer 11
    ・Internet Explorer 10
    ・Internet Explorer 9
    ・Internet Explorer 8
    ・Internet Explorer 7
